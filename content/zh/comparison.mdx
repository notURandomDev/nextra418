# 模版对比

Nextra 拥有一个充满活力的生态系统，提供了许多优秀的启动模板。**Nextra418** 的目标是提供一个 **极简、现代且生产可用** 的基础架构。

为了帮助你决定这个模板是否适合你，我们将其与其他流行的 Nextra 模板进行了对比。

## 模版对比

| 模版 | 架构 | 核心特性 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **Nextra418** (本项目) | **App Router** | 极简主义, i18n, Pagefind 搜索 | 追求简洁、高性能起点的开发者。 |
| [官方模版](https://github.com/shuding/nextra-docs-template) | Pages Router | 官方标准, 基础配置 | 习惯传统 Pages 目录结构的用户。 |
| [Docs Starter](https://github.com/phucbm/nextra-docs-starter) | App Router | Tailwind v4, 丰富的组件示例 | 想要内容丰富、开箱即用的 UI。 |
| [Landing Template](https://github.com/gfazioli/next-app-nextra-template) | App Router | Landing Page, Mantine UI, Jest | 需要营销落地页 + 文档的项目。 |
| [Personal Website](https://github.com/namnguyenthanhwork/nextra-personal-website-template) | App Router | 博客 + 文档, shadcn/ui | 个人作品集、博客与文档集成。 |
| [SWR Site 示例](https://github.com/shuding/nextra/tree/main/examples/swr-site) | App Router | 深度 i18n, 复杂参考 | 学习 Nextra 高级用法和边缘案例。 |

## 为什么选择 Nextra418？

### 1. 极简主义哲学
我们认为启动模板应该是 **“起点”**，而不是终点。我们剔除了所有非必要的代码，让你能够完全掌控项目的架构和样式。

### 2. 现代架构
虽然许多模板仍在使用 `pages` 目录，但 Nextra418 完全基于 **Next.js App Router** (Next.js 16) 和 **React 19** 构建，充分利用了 Server Components 的性能优势。

### 3. 开箱即用的 i18n
Nextra418 遵循 Nextra 4 官方推荐的 i18n 方案，并进行了精简化处理，使其在生产环境中既稳健又易于维护。

### 4. 无后端的极速搜索
内置 **Pagefind**，让你无需配置外部索引服务或数据库，即可获得毫秒级的全文搜索体验。
